# Lista mínima de Casos E2E (20)

1) Login OK / 2) Login KO
3) Dashboard visible y KPIs > 0
4) Crear paciente y aparece en lista
5) Editar paciente y persiste
6) Inhabilitar / habilitar paciente y filtro aplica
7) Eliminar paciente y se remueven sesiones asociadas
8) Crear sesión y aparece en listado
9) Editar sesión
10) Eliminar sesión
11) Filtro sesiones por fecha
12) Filtro sesiones por paciente
13) Notificación éxito en CRUD
14) Notificación error (validación)
15) Health 200 y Metrics 200 con campos esperados
16) Endpoint admin protegido por rol
17) CSP strict sin violaciones en consola
18) Export/Descarga (si aplica)
19) Mantenimiento bloquea writes (503)
20) Rate‑limit auth en emisión de token
